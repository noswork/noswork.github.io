# Changelog

All notable changes to NOS SITE will be documented in this file.

## [2.0.0] - 2025-10-31

### 🎉 重大更新

完全重新設計的主頁，提供更專業、更現代的使用者體驗。

### ✨ 新增功能

#### 1. 🎨 主題系統
- **明亮/昏暗模式切換**：點擊導航欄的主題按鈕即可切換
- **自動記憶偏好**：使用 localStorage 保存用戶選擇
- **流暢過渡動畫**：主題切換時的平滑視覺效果
- **完整色彩系統**：使用 CSS 變數實現的現代化色彩方案

#### 2. 🌍 多語言支援
- **繁體中文（zh-TW）**：預設語言
- **簡體中文（zh-CN）**：完整翻譯
- **英文（en）**：完整翻譯
- **日文（ja）**：完整翻譯
- **動態切換**：無需重新載入頁面
- **偏好記憶**：自動保存用戶選擇的語言

#### 3. 🎯 自定義 SVG 圖標系統
- 移除所有 emoji 圖標，改用專業的 SVG 圖標
- 完全可自定義的顏色和大小
- 支援主題色彩切換
- 零外部依賴，更快的載入速度
- 向量圖形，完美縮放

##### SVG 圖標列表
- Logo 圖標（方形設計）
- 語言切換圖標（地球儀）
- 主題切換圖標（太陽/月亮）
- 詭計少女計算器（日曆圖標）
- 迷宮遊戲（迷宮圖標）
- Prompt 優化器（星星圖標）
- 簡繁轉換工具（文件圖標）
- 各種功能圖標（剪貼板、圖表、書籍等）

#### 4. 📱 全新響應式設計
- **桌面版（>1024px）**：
  - 4 欄工具卡片網格
  - 2 欄功能介紹布局
  - 完整的視覺效果

- **平板版（768px-1024px）**：
  - 自適應 2 欄布局
  - 優化的間距和字體大小

- **手機版（<768px）**：
  - 漢堡選單導航
  - 1 欄卡片布局
  - 觸控優化的按鈕尺寸
  - 側邊滑出選單

#### 5. 🎬 動畫與互動效果
- **滾動動畫**：使用 Intersection Observer 實現元素淡入
- **導航欄效果**：
  - 滾動時自動添加背景和陰影
  - 向下滾動時智能隱藏
  - 向上滾動時重新顯示
- **卡片互動**：
  - 懸停時上浮效果
  - 點擊波紋反饋
- **平滑滾動**：錨點連結的平滑過渡
- **視差效果**：首頁的輕微視差動畫

#### 6. 🎨 全新設計系統
- **現代化色彩**：
  - 主色：Indigo (#6366f1)
  - 輔色：Purple (#8b5cf6)
  - 強調色：Pink (#ec4899)
- **統一的間距系統**：從 0.5rem 到 6rem 的標準化間距
- **圓角系統**：4 種標準化圓角尺寸
- **陰影系統**：4 級陰影深度
- **過渡動畫**：統一的動畫曲線和時長

### 🗑️ 移除內容

#### 刪除的目錄
- ❌ `tga/` - 東京喰種地圖工具（不常用）
- ❌ `tlotsas/` - 舊版計算器（已被 TrickCal 取代）
- ❌ `src/mcp/` - sequential-thinking 開發工具
- ❌ `generators/` - novel 和 talkshit 生成器

#### 移除的數據
- ❌ 所有虛假的統計數據（如「105+ 角色」）
- ❌ 捏造的用戶數量和使用統計
- ✅ 改用真實、可驗證的功能描述

### 🔧 技術改進

#### 前端架構
- **模組化 JavaScript**：使用 ES6 模組系統
- **CSS 變數**：完整的設計令牌系統
- **語義化 HTML**：改進的 HTML 結構和無障礙性
- **性能優化**：
  - 使用 Intersection Observer 替代滾動監聽
  - CSS transform 動畫（GPU 加速）
  - 節流和防抖優化

#### 代碼品質
- **無 Linter 錯誤**：所有文件通過 linter 檢查
- **一致的命名**：統一的變數和函數命名規範
- **完整註釋**：關鍵函數和複雜邏輯的詳細註釋
- **模組化結構**：清晰的文件和功能分離

### 🚀 部署指南

1. **提交更改**：
   ```bash
   git add .
   git commit -m "v2.0.0: 完全重新設計主頁，添加主題和多語言支援"
   git push
   ```

2. **DNS 配置**（如尚未配置）：
   - 添加 A 記錄指向 GitHub Pages IP
   - 或添加 CNAME 記錄指向 `noswork.github.io`

3. **等待部署**：GitHub Pages 自動部署通常需要 1-5 分鐘

4. **驗證**：
   - 訪問 https://nossite.com
   - 測試所有功能
   - 檢查不同裝置和瀏覽器
